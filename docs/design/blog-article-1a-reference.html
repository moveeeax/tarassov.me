Zero-downtime Kubernetes upgrades on three clouds

kubernetes terraform sre

Upgrading a cluster that serves live traffic is mostly a matter of ordering. The control plane goes first, then node pools — one surge node at a time, with pod disruption budgets doing the actual safety work.

1. Drain with a budget

A PodDisruptionBudget is the only thing standing between a rolling node upgrade and a full outage of a two-replica service:

apiVersion: policy/v1
kind: PodDisruptionBudget
metadata:
  name: api-pdb
spec:
  minAvailable: 1
  selector:
    matchLabels:
      app: api
